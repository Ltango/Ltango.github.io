Recent Projects

  • The cyber network capture generator was my senior design project at Iowa State. This project was under the constraints of utilizing free and open-sourced software. There are three major components, the network generator, the network capture, and the web application. The workflow begins with the push of a button - set up a scenario with 2 virtual machines routed through a machine hosting pfsense to the outside web. I was responsible for creating scripts that generate traffic utilizing a selenium web crawler, SSH between machines on the network, portscans, and remote execution of code. All of this traffic is recorded in the network and sent to the web application for furhter investigation after the scenario has finished. The security research aspects to this are infinite as any malware can be introduced into the system safely for further study.

  • Another application I helped develop at my internship doing IT security was a script that ran 24/7. Several thousand suspicious applications would be flagged by one of the companies security solutions Sentinel One. I wrote a script that would take those threats, look up their file hash in Virus Total or submit a request to review the file hash. The Virus Total API would then return a value of how dangerous the application was, and anything above a certain threshold the script would make calls to the Sentinel One API to automatically kill, quarantine, and resolve the threat. The file hash would then be recorded to automatically resolve the threat the same way in the future to minimize calls to the VirusTotal API. Another aspect of this script would be to automatically check all embedded URLs in our Jira ticket system. Using selenium to find final URL redirects, that URL would then be uploaded to VirusTotal for review for their API. The score would be returned, and all of this information would be written as an internal comment in Jira. If scores were above a certain threshold then it would be blocked network wide by mimecast, and through DNS lookups. This proved to be extremely effective as I would be closing 100 tickets to other team members 50 tickets every week despite working half the hours. There was automatic error handling so that any errors would be recorded and I could review them individually while the script ignores the ticket and continues running. Built in logging would keep records on the server for about a month.
